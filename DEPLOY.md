# Deploying BizNova to Render

This app deploys as a single Render Web Service with a 1 GB persistent disk
that holds the SQLite database (`/var/data/biznova.db`).

## 1. Push the code to GitHub

```powershell
cd C:\Users\bonni\Documents\Thapelo
git init
git add .
git commit -m "Initial BizNova commit"
# Create an empty repo on github.com first, then:
git branch -M main
git remote add origin https://github.com/<your-username>/biznova.git
git push -u origin main
```

`.env` and `biznova.db` are already in `.gitignore` so secrets stay local.

## 2. Create the service on Render

1. Sign in at <https://dashboard.render.com>.
2. Click **New +** → **Blueprint**.
3. Connect your GitHub account and pick the `biznova` repo.
4. Render reads `render.yaml` and proposes a `biznova` web service with a
   1 GB disk mounted at `/var/data`. Click **Apply**.

## 3. Set the secret env vars

In the new service's **Environment** tab, fill in:

| Key | Value |
|-----|-------|
| `OPENAI_API_KEY` | your real OpenAI key |
| `SMTP_HOST` | e.g. `smtp.gmail.com` (optional — without it, OTPs print to logs) |
| `SMTP_USER` | SMTP username |
| `SMTP_PASS` | SMTP app password |
| `SMTP_FROM` | the From: address |

`FLASK_SECRET_KEY` is auto-generated by Render. `BIZNOVA_DB_PATH`,
`FLASK_ENV=production`, and `PYTHON_VERSION` are already set by the blueprint.

Click **Save Changes** → Render redeploys.

## 4. Visit the live app

When the deploy turns green, open
`https://biznova.onrender.com` (or whatever subdomain Render assigns).

The login page should load. Sign up with any email — if SMTP isn't
configured, the dev OTP appears in the **Logs** tab and in the API response.

## Notes

- **Cold starts**: free tier sleeps after 15 min of no traffic. First
  request after sleep takes ~30 s.
- **Persistent disk**: free tier disks survive deploys but **not** plan
  changes; back up `biznova.db` if you ever resize.
- **Sessions**: cookies are `Secure` + `HttpOnly` + `SameSite=Lax` in
  production (auto-detected via `RENDER` or `FLASK_ENV=production`).
- **Logs**: Render → your service → **Logs** tab to see OTP printouts and
  request lines.
- **Custom domain**: Settings → Custom Domains → add your domain and
  follow the DNS instructions.
